Section 03A

Core Concept Crash Course

Learn the foundations of Retrieval-Augmented Generation (RAG) and the LangChain orchestration framework before dive-building.

Retrieval-Augmented Generation (RAG)

LLMs are smart but frozen in time and blind to private data. RAG acts as an "open-book exam" — retrieving relevant context from a database and placing it into the prompt before generating an answer.

1
Data Ingestion (Chunk & Embed)

Documents are split into smaller paragraphs (chunks), converted into mathematical vectors (embeddings), and saved in a Vector Database.

2
Similarity Retrieval

When a user asks a question, the query is embedded, and a similarity search matches it with the most semantically relevant document chunks.

3
Context-Augmented Output

The query and fetched chunks are bundled into a prompt template. The LLM reads this "context" and generates an accurate, hallucination-free answer.

Why RAG wins: (1) 100x cheaper, (2) Updates instantly when files change, (3) Includes direct citations/sources, (4) Strict user-access controls.

LangChain vs Raw Building

The roadmap teaches you to build raw first. Why? Look at how chains are created:

# LangChain LCEL Chain:\nchain = prompt | model | StrOutputParser()\nresult = chain.invoke({"query": "Hello"})

LCEL is beautiful, but debugging inner layers, managing timeouts, logging custom spans, and applying granular cost control is much simpler when writing standard Python await calls.

Key Interview Prep Qs ▼
  • What is the difference between concurrency and parallelism in Python, and how does the GIL affect async code?Asyncio uses cooperative multitasking (single thread) to yield control during I/O blockages. Parallelism uses multiple processes. The GIL prevents multiple native threads from executing Python bytecodes concurrently, making asyncio ideal for high-concurrency I/O (like prompt streaming) but weak for CPU-heavy jobs.
  • Why should you use Pydantic v2 over standard classes/dictionaries in production LLM gateways?Pydantic v2 is written in Rust, which operates data validation 5x to 15x faster than pure Python. It guarantees strict type checks, handles dynamic nested structures returned from LLMs, and formats responses with clear JSON schemas automatically.

For user engagement, sub-500ms TTFT is essential. Long completion times are tolerable if the text starts streaming immediately.

Key Interview Prep Qs ▼
  • What is an HNSW index in vector search, and how does it compare to IVFFlat in production scaling?HNSW (Hierarchical Navigable Small World) builds a multi-layer graph of vectors, allowing extremely high-recall search in sub-millisecond times at the cost of higher memory usage. IVFFlat splits vectors into clusters, using less memory but sacrificing recall at scale.
  • Why does simple recursive character chunking fail for highly structured data like financial reports?Financial tables and metrics require full horizontal context. Slicing purely by characters breaks rows and columns across chunks, causing the LLM to receive disconnected numbers, leading to hallucination of balances and totals.

Key Interview Prep Qs ▼
  • What is Reciprocal Rank Fusion (RRF) and why is it essential for production-grade search systems?RRF combines search results from multiple algorithms (like keyword BM25 and semantic vector similarity) without needing calibrated scores. It ranks items based on their reciprocal positions across both lists, capturing both exact terminology and conceptual meaning.
  • How does HyDE (Hypothetical Document Embeddings) improve RAG retrieval accuracy?Users often write short, keyword-sparse queries. HyDE prompts an LLM to generate a 'fake' or hypothetical ideal answer first. We embed this detailed answer and use it to search the vector database, vastly improving retrieval recall.

Key Interview Prep Qs ▼
  • What is the 'agent loop of death' and how do you protect production systems from it?It's an infinite loop where an agent repeatedly calls tools with the same failing input. We protect against this by enforcing strict maximum iteration limits (e.g., max 10 steps), timeout thresholds, and max budget caps per execution.
  • Why is LangGraph preferred over simple LangChain chains for complex agent workflows?Simple chains are linear and struggle with loops. LangGraph models workflows as cyclical state charts. You can explicitly define loops, state updates, conditional paths, and easily save/restore state at any node for human approvals.

  • How do you design a robust 'LLM-as-a-Judge' evaluation prompt that avoids bias?Specify clear evaluation criteria (e.g., rating scale from 1-5 with strict definitions for each score), provide few-shot examples of correct scoring, force the model to output its reasoning before the score, and calibrate its outputs against human-graded datasets.

An eval pipeline measures semantic consistency, groundedness, and hallucination metrics on a test suite of 100+ scenarios, catching regressions in prompts or models.

Key Interview Prep Qs ▼
  • Explain how semantic caching differs from traditional Redis key-value caching in LLM gateway design.Traditional caching requires an exact character match. Semantic caching embeds the incoming prompt, performs a vector similarity search on historically cached prompts, and returns the cached answer if the similarity score is above a threshold (e.g., 0.95), saving time and API costs.
  • How do you handle fair-use rate limiting in a multi-tenant LLM platform where users share a single OpenAI API key?We implement token buckets per tenant-id. Each request deducts tokens from the bucket based on input + output estimates. We reject requests at the gateway level once the bucket is empty, protecting our upstream keys from getting rate-limited or exhausted.

Key Interview Prep Qs ▼
  • When should a startup shift from commercial APIs like OpenAI to open-source models like Llama-3 served on their own GPUs?When query volume is high and constant, data privacy is legally required, or the tasks are simple (like text classification).   • How does vLLM achieve high throughput in model serving?vLLM uses PagedAttention. Traditional LLM serving wastes memory storing Key-Value states in contiguous blocks. PagedAttention divides the KV cache into logical blocks (like pages in operating systems), completely eliminating memory fragmentation and allowing 10-30x higher throughput.

Key Interview Prep Qs ▼
  • How do you handle interruptions in a real-time full-duplex Voice AI application?We monitor the incoming WebSocket audio channel from the user. When streaming STT detects new user voice while the TTS is speaking, we immediately send a cancellation token to the TTS stream, clear the audio queue, and recalculate the state based on the user's new words.
  • What is the architectural bottleneck for sub-800ms conversational audio response, and how do you solve it?The bottleneck is serial execution (STT finished -> LLM finished -> TTS starts). We solve this by streaming everything: we stream STT text -> stream LLM tokens directly to TTS -> stream TTS audio chunks back to WebSockets. This overlaps all steps in parallel, cutting response latency in half.

Key Interview Prep Qs ▼
  • Why is version pinning critical for LLMs, and how do model updates break production pipelines?LLM updates (e.g. OpenAI updating gpt-4o under the hood) can change response formats, tone, and system instruction adherence. If a prompt depends on exact JSON structures, the updated model might skip key properties, instantly breaking API integrations. We must lock exact snapshots (e.g. `gpt-4o-2024-08-06`) and verify updates thoroughly before deployment.
  • How do you secure LLM API keys in multi-tenant or serverless deployments?We store API keys in secure, encrypted vaults (AWS Secrets Manager, HashiCorp Vault, or environment configs). Section 06

Tech Stack Map

The complete stack, organized by layer. Click any layer to expand. Build in this order.

🐍 Language + Runtime
Python 3.13+asyncioPydantic v2uv

Python 3.13+ with strict typing (mypy). Pydantic v2 for all data models. asyncio + httpx for async I/O. uv for packaging (faster than pip, handles virtual envs correctly). FastAPI for all APIs — not Flask, not Django, not Express.

🤖 LLM Providers + Gateway
OpenAIAnthropicGeminiLiteLLM

Learn all three major providers. Use LiteLLM as a unified gateway — single interface, model fallback, cost tracking. For local inference: Ollama (development), Groq (production low-latency), Together AI (open-source models).

🗄️ Databases + Vector Stores
PostgreSQL + pgvectorRedisQdrant

PostgreSQL with pgvector is your primary vector store — no new infra if you're already on Postgres. Redis for semantic caching and session state. Qdrant for high-scale dedicated vector search. SQLAlchemy async ORM for PostgreSQL. Alembic for migrations.

🕸️ Agents + Orchestration
LangGraphRaw function callingCelery/BullMQ

Build raw function calling first — understand the mechanics. Use LangGraph for stateful agent orchestration. Avoid LangChain chains (leaky abstractions). For async AI job queues: Celery (Python) or BullMQ (Node). For agent state persistence: Redis or Postgres.

📡 Evals + Observability
RagasLangSmithDeepEvalOpenTelemetry

Ragas for RAG evaluation (faithfulness, relevance, groundedness). DeepEval for broader LLM evals. LangSmith for tracing and observability. OpenTelemetry for custom spans. GitHub Actions for CI eval gates. These are non-negotiable for production AI systems.

☁️ Infrastructure + Deployment
RailwayFly.ioAWS Lambda/S3/SQSDocker

Railway or Fly.io for primary deploys (fast, cheap, no Kubernetes). AWS for heavy lifting: S3 for document storage, Lambda for async inference, SQS for job queues. Docker everywhere. GitHub Actions for CI/CD with eval gates. Not Kubernetes — that's premature at this scale.

Weekly Schedule

How to structure 12–16 hours per week as a working engineer. Assumes you have a full-time job. The schedule is battle-tested — the blocks matter.

Weekday Morning Blocks (Mon, Wed, Fri) — 1.5 hrs each
6:00–7:30
Deep study or project build. No meetings, no messages. Phone on airplane mode. This is the highest-quality cognitive block of the day. Use it for hard concepts or focused builds — not for setup or admin.
Weekend Sessions (Sat + Sun) — 4 hrs each
9:00–11:00
Primary build session. Continue or start a new project milestone. This is when the real output happens — 2 hours of uninterrupted work equals 4 hours of interrupted work.
11:00–12:00
Lunch + light reading. Documentation, blog posts, architecture diagrams. Passive intake while resting.
13:00–15:00
Second build block or video learning. Video learning goes here, not in prime morning slots. Complete the week's milestone if behind.
Weekday Evening (Tue, Thu) — optional 45 min
21:00–21:45
Concept review or reading. Low-intensity: flashcard review, documentation reading, note-taking. Not active builds — your brain is tired. Use this to consolidate, not create.

Real talk on hours: 12 hrs/week is the minimum for meaningful progress. 16+ hrs/week is what gets you to 8 months instead of 12. Missing a weekend session is fine — two consecutive missed weekends means you've lost momentum. Recovery time doubles.

Section 09

Reality Check

The uncomfortable truths most roadmaps omit. Read when you feel on track — not just when struggling.

Delusions That Waste Time

  • "I need to understand transformers mathematically before building." Build first. Math comes after.
  • "More certificates = more credibility." Zero AI companies care. They look at GitHub and what you've shipped.
  • "LangChain will abstract away the complexity." LangChain abstracts away your understanding. Build raw first.
  • "Fine-tuning will make my system better." RAG or better prompts solve 90% of problems.
  • "If I learn one more framework, I'll be ready." You're ready when you have deployed projects with real architecture decisions.

What Creates Leverage

  • Deployed projects with real metrics — not GitHub repos
  • Ability to explain RAG tradeoffs in 2 minutes without slides
  • An eval pipeline that actually measures quality
  • Multi-tenant architecture knowledge
  • Cost optimization story — saving 50% on LLM costs is a business impact story
  • 3–4 solid public technical posts showing depth

What "Job-Ready" Actually Means: You can design a production RAG system from scratch in 45 minutes. You know why you'd choose hybrid search over pure vector. You have 2+ live deployed projects with architecture decisions documented. You can explain where your projects would fail at 10× scale and how to fix it.
